Current

During the Advanced Software Lab course at San Francisco State University, I have become familiar with several technologies. The following is a current summary of the web-design technologies I have had experience with:

  • HTML - Basic Web Design
  • Bootstrap - Advanced Web Design
  • JavaScript - Changing background color
  • JQuery - Button Hide/Show response
  • PHP - Databases with SQL, $_POST Confirmation
  • SQL - Creating and inserting into a table with PHP
  • Databases - Printing from and inserting into a DB with PHP and SQL
  • Python - Streaming from Twitter using Tweepy API
  • VCS - More specifically SVN and how to collaborate using it

More specifically, I gained knowledge of basic HTML involving any kind of formatting of text, insertion of forms and the importance of using CSS to modify the attributes of the code to make the work-process less time-consuming. I know of bootstrap in relation to more advanced web-design including, but not limited to, how to design a site that is responsive to mobile devices. This site uses the default settings for mobile devices, meaning that the width of this site adapts to the screen-width of the device it is viewed on. The list-group above is also made with bootstrap in order to benefit from the design the design-theme that bootstrap adds.

Javascript was used in the Demonstrations section so that on_click of the change background event calls a function that changes the background color of the page. jQuery was used for the hide-show buttons in the Demonstrations sections to tie the on_click event of the button together with the ready() function from the jQuery library that hides/shows a paragraph with a specific id. The id is selected with a Javascript selector

PHP is used to respond to the clicking of the submit button with the superglobal _$POST variable and was also used for a lab exercise involving databases. For security reasons, the result of the work has not been posted. The PHP code in mention simply allows any user to post a quote with an author and the quote and author will then be inserted automatically into a table in a database using SQL. If the table does not exist, it is created.

Another PHP-project that has also been omitted from this site for the same security concerns as above allows the printing of a table in a database onto a page. VCS is not displayed on this site, but I have experience with the VCS called SVN and how to upload and merge files without overwriting changes that another user has made. The files were created with VIM in a UNIX shell set up to communicate with the class server through the Secure Shell Protocol.

I also used Tweepy API to livestream any twitter feed with a given # in real-time by using Python in VIM on an Amazon Web Services Server, because that server had the correct packages installed to run the API. For concrete examples of the above-mentioned, see the Demonstrations section.
